# opstack-doctor v0.1.13 Release Notes

`v0.1.13` adds a release metadata preflight so version drift is caught before tagging.

## Highlights

- Add `make release-preflight`.
- Wire release preflight into `make release-check`.
- Verify `VERSION` matches `cmd/opstack-doctor/main.go`.
- Verify README release snippets use the same version.
- Verify the latest dated changelog heading matches `VERSION`.
- Verify `docs/releases/v$(cat VERSION).md` exists and mentions both GHCR image tags.
- Add unit coverage for version drift, README drift, release-note tag coverage, and `Unreleased` changelog handling.

## Operator Impact

- Release artifacts and docs are less likely to disagree about the advertised version.
- Maintainers get a fast local check before opening a release PR or pushing a tag.
- Feature branches can still keep an `Unreleased` changelog section above the latest released heading without breaking the preflight for the current release.

## Container Images

```sh
ghcr.io/cruzmolina/opstack-doctor:v0.1.13

```

## Validation

Before tagging, maintainers should run:

```sh
make release-check
make docker-build
make docker-smoke
```
