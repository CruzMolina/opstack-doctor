# opstack-doctor v0.1.15 Release Notes

`v0.1.15` adds deterministic fixture reports for support, documentation, regression testing, and alert-pipeline rehearsals.

## Highlights

- Add `opstack-doctor fixture healthy|warn|fail` with human, JSON, and Prometheus output modes.
- Check in stable JSON examples for healthy, warning-only, and failing diagnostic reports.
- Keep fixture outputs entirely local and deterministic; the command does not start mock servers or contact RPC/metrics endpoints.
- Include derived Prometheus metrics in fixture output, including execution lag and topology follower lag, so alert ingestion can be rehearsed without a live fleet.
- Add fixture command coverage to shell completion, command tests, smoke checks, Docker smoke checks, README, and contributor docs.

## Operator Impact

- Operators can share known-good report shapes in support threads without exposing infrastructure details.
- Alerting and automation pipelines can test JSON and Prometheus parsing against stable examples before live endpoint access exists.
- The existing `demo` command remains available for end-to-end mocked checks through the normal engine; `fixture` is the deterministic no-network path.
