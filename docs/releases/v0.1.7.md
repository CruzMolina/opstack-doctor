# opstack-doctor v0.1.7 Release Notes

`v0.1.7` expands generated alert rule tests for native interop service metrics.

## Highlights

- Add `promtool test rules` coverage for native op-supervisor alert templates.
- Add `promtool test rules` coverage for native op-interop-mon alert templates.
- Validate firing behavior for supervisor down, access-list verification failures, risky interop monitor message statuses, and terminal status changes.
- Keep the existing doctor-exported alert firing tests from `v0.1.6`.

## Operator Impact

- Native op-supervisor and op-interop-mon alert templates now have representative firing-behavior tests.
- This further reduces the chance that generated interop alert rules parse successfully but fail to fire for expected metric inputs.
- Site-specific Prometheus label edits still need validation against the operator's own Prometheus environment.
