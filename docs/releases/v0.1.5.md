# opstack-doctor v0.1.5 Release Notes

`v0.1.5` improves the production-readiness path for generated alert rules and scheduled Prometheus exports.

## Highlights

- Add doctor-exported Prometheus alert templates for interop dependency, op-supervisor, and op-interop-mon readiness findings.
- Add a representative Prometheus export example for interop findings from scheduled `opstack-doctor export metrics` runs.
- Document selectors that alert on doctor-exported interop findings when Prometheus does not directly scrape every internal interop endpoint.
- Add pinned Prometheus `promtool` validation for the checked-in generated alert rules.
- Run `promtool` validation in CI and as part of `make release-check`.

## Operator Impact

- Operators can alert from scheduled doctor output through `opstack_doctor_finding` series for interop dependency, op-supervisor, and op-interop-mon readiness findings.
- The checked-in generated alert example now gets PromQL syntax validation before merge and release.
- Site-specific Prometheus label edits still need local validation with the operator's own Prometheus tooling.
