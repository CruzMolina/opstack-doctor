# opstack-doctor v0.1.12 Release Notes

`v0.1.12` makes the generated config schema a tested operator-facing contract.

## Highlights

- Add `opstack-doctor generate schema --out doctor.schema.json`.
- Check in `examples/doctor.schema.json` for editor and CI integrations.
- Add schema parity tests so the checked-in schema stays generated from the CLI.
- Add schema contract tests that validate `examples/doctor.example.yaml`.
- Add intentionally invalid fixtures for missing required fields, invalid roles, invalid URL schemes, and unexpected properties.
- Add `make schema-check` and run it as part of `make release-check`.

## Operator Impact

- Operators can generate a JSON Schema from the installed binary and use it in editor validation or config-review CI.
- The public example config is now validated against the checked-in schema before release.
- Config-shape drift is easier to catch before it reaches packaged binaries or container images.
- Live topology validation still belongs to `opstack-doctor validate` and `opstack-doctor check`; the schema is intentionally static and endpoint-free.
