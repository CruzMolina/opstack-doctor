# opstack-doctor v0.1.10 Release Notes

`v0.1.10` adds offline config validation for operator configs.

## Highlights

- Add `opstack-doctor validate --config doctor.yaml`.
- Support `--output human` and `--output json`.
- Support `--fail-on fail`, `--fail-on warn`, and `--fail-on none`.
- Reuse the same structured config findings between `validate` and live `check`.
- Add validate smoke coverage to local and Docker release checks.

## Operator Impact

- Operators can lint config shape and topology intent without contacting RPC or metrics endpoints.
- CI systems can gate on config failures before running live diagnostics.
- Teams can use stricter warning gates with `--fail-on warn` while still rendering reports with `--fail-on none`.
