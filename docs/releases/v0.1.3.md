# opstack-doctor v0.1.3 Release Notes

`v0.1.3` hardens the metric-checking foundation added in `v0.1.2`.

## Highlights

- Add a redacted Prometheus fixture corpus for op-node and proxyd healthy, warning, failure, missing-optional, and label-variant cases.
- Add table-driven regression tests that assert stable finding IDs and severities from those fixtures.
- Document how contributors should capture, minimize, and redact metrics fixtures safely.
- Reduce noise for proxyd deployments that do not expose version-specific backend probe or backend latency metrics by reporting those missing metrics as `info`.

## Operator Impact

- `proxyd_up != 1` remains a failure.
- Missing consensus-aware proxyd count/block gauges remain warnings for consensus-aware endpoints.
- Missing version-specific proxyd backend probe and latency metrics are now informational findings, which should reduce false urgency for older or differently configured proxyd deployments.
