# opstack-doctor v0.1.2 Release Notes

`v0.1.2` adds proxyd-native metric checks on top of the `v0.1.1` declared routing topology checks.

## Highlights

- Check `proxyd_up` for process health.
- Check backend probe health, degraded backends, banned consensus backends, backend sync state, and peer-count metric presence.
- Check consensus latest/safe/finalized gauges and serving consensus backend counts.
- Check CL/source-tier consensus counters for bans, output-root disagreement, and missing pin candidates.
- Check backend request latency quantiles against `thresholds.max_rpc_latency_seconds`.
- Generate proxyd-native Prometheus alert templates and runbook incident sections.

## Operator Impact

- Existing configs remain valid.
- proxyd metrics are still treated conservatively: missing version-specific metrics are usually WARN or INFO, while `proxyd_up != 1` is a FAIL.
- Counter checks report nonzero lifetime counters as warnings; production Prometheus alerts use `increase(...)` where appropriate.
