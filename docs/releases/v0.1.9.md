# opstack-doctor v0.1.9 Release Notes

`v0.1.9` hardens generated runbook regression coverage.

## Highlights

- Add golden-file regression coverage for generated Markdown runbooks.
- Add command-level parity coverage proving `examples/runbook.example.md` is generated from `examples/doctor.example.yaml`.
- Check in a generated sample runbook for operators reviewing the sample config.
- Document the runbook fixture and public example refresh workflow.

## Operator Impact

- Generated runbook changes are now easier to review because checklist, incident response, topology, and reference-link changes appear in checked-in fixture diffs.
- Operators can inspect a generated runbook example without running the CLI first.
- The public runbook example is guarded against drifting away from the sample operator config.
