# opstack-doctor v0.1.4 Release Notes

`v0.1.4` adds optional interop-specific metrics readiness checks for operators experimenting with or preparing dependency-set observability.

## Highlights

- Add optional op-supervisor metrics readiness checks for process up status, local/cross safety refs, expected chain coverage, access-list verification failures, log DB metrics, and RPC metric presence.
- Add optional op-interop-mon metrics readiness checks for process up status, executing-message status gauges, terminal status changes, and block-range metric presence.
- Extend generated Prometheus alert templates and Markdown runbooks with op-supervisor and op-interop-mon incident sections.
- Add redacted op-supervisor and op-interop-mon Prometheus fixtures with regression tests for healthy and risky scenarios.
- Document the new `interop.supervisor` and `interop.monitor` config blocks.

## Operator Impact

- These checks are opt-in through `interop.supervisor.metrics` and `interop.monitor.metrics`; existing configs keep working without changes.
- `op_supervisor_*_up != 1` and `op_interop_mon_*_up != 1` are failures when those endpoints are configured.
- Missing supervisor refs, missing expected chain coverage, missing monitor message status, and risky message statuses are reported as warnings.
- The checks remain read-only readiness diagnostics. They do not validate cross-chain message execution or prove full op-supervisor protocol correctness.
