# opstack-doctor v0.1.14 Release Notes

`v0.1.14` improves source/light-node safe-head diagnostics and tightens release CI.

## Highlights

- Parse explicit L2 `op_node_default_refs_number` safe/finalized/unsafe refs across split labels such as `type="safe", layer="l2"`.
- Parse combined, compact, and case-varied labels such as `ref="l2_safe"`, `ref_name="L2-FINALIZED"`, and `type="unsafe_l2"`.
- Treat bare refs such as `ref="safe"` as ambiguous rather than silently assuming they are L2.
- Ignore L1 refs when choosing safe-head values for topology comparisons.
- Include the matched source and follower safe-ref series in topology lag evidence.
- Add fixture coverage for op-node refs label variants.
- Run `make release-preflight` as an explicit GitHub Actions CI step.

## Operator Impact

- Source/light-node topology checks can recognize more real-world op-node refs label shapes.
- Safe-head lag findings now show the exact matched metric series, which makes dashboard and scrape-label debugging faster.
- Ambiguous metrics remain conservative instead of producing false confidence.
- Release metadata drift is now caught directly in CI as well as local release checks.
