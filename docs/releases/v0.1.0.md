# opstack-doctor v0.1.0 Release Notes

`v0.1.0` is the first MVP release of `opstack-doctor`, a read-only diagnostic CLI for OP Stack and Superchain operators.

## Highlights

- Validate op-geth to op-reth migration readiness with side-by-side read-only RPC checks.
- Check specialized source/light-node topology intent and follower lag.
- Check basic interop dependency endpoint readiness.
- Parse op-node Prometheus metrics and emit structured findings.
- Export doctor findings as Prometheus text metrics for scheduled monitoring.
- Generate Prometheus alert rules and Markdown runbooks.
- Ship as release archives and GHCR container images.

## Install

Download a platform archive from the GitHub release, verify `SHA256SUMS`, and run:

```sh
./opstack-doctor version
./opstack-doctor demo --scenario healthy
```

Container images are published as:

```sh
ghcr.io/OWNER/REPO:v0.1.0
ghcr.io/OWNER/REPO:0.1.0
```

Replace `OWNER/REPO` with the published repository path.

## Notes For Operators

- The tool is read-only. It does not send transactions and does not require private keys.
- RPC URLs are redacted before being printed in findings.
- Tests use mocked endpoints and do not hit public RPC services.
- Generated alert rules are templates; validate label selectors against your Prometheus series before production use.

## Known Limitations

- Interop checks do not validate cross-chain messages or op-supervisor behavior.
- RPC comparison is sampled and not exhaustive.
- Proxyd topology checks and Grafana dashboard generation are future work.
