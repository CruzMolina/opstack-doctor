# opstack-doctor v0.1.1 Release Notes

`v0.1.1` is a release-hygiene and routing-readiness update for `opstack-doctor`.

## Highlights

- Add Apache-2.0 licensing to release-ready source archives.
- Add proxyd/routing topology checks for declared deriver and edge endpoints.
- Export proxyd head lag as Prometheus text metrics for scheduled doctor runs.
- Generate proxyd-related Prometheus alert templates and runbook sections.
- Update GitHub Actions to current major versions and add Dependabot coverage.

## Operator Impact

- Existing configs remain valid. If `proxyd.enabled` is omitted or false, proxyd checks are disabled unless the config already expresses a specialized source/light-node topology, in which case doctor emits a warning to configure production routing checks.
- New `proxyd` config fields are declarative. `opstack-doctor` does not read private proxyd TOML and does not prove every backend is live inside proxyd; it checks externally visible RPC/metrics endpoints and declared backend heads.
