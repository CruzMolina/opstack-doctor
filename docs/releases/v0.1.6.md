# opstack-doctor v0.1.6 Release Notes

`v0.1.6` adds firing-behavior tests for representative generated Prometheus alerts.

## Highlights

- Add `promtool test rules` fixtures for generated alerts.
- Cover representative doctor-exported interop readiness warnings, execution candidate lag, and proxyd readiness failures.
- Add `make promtool-test`.
- Run `promtool-test` in CI and as part of `make release-check`.
- Document the difference between Prometheus rule syntax validation and alert firing tests.

## Operator Impact

- Generated alert templates now have both syntax validation and representative firing-behavior validation before merge and release.
- This reduces the chance that a rule parses but never fires for the intended `opstack-doctor` export series.
- Site-specific Prometheus label edits still need validation against the operator's own Prometheus environment.
