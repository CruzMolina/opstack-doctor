# opstack-doctor v0.1.8 Release Notes

`v0.1.8` hardens generated alert-rule regression coverage.

## Highlights

- Add golden-file regression coverage for generated alert YAML.
- Add an `UPDATE_GOLDEN=1` workflow for intentional alert generator updates.
- Validate both the public example rules and golden alert rules with `promtool`.
- Add command-level parity coverage proving `examples/prometheus-rules.example.yaml` is generated from `examples/doctor.example.yaml`.
- Add negative `promtool test rules` cases proving representative alerts stay quiet on healthy inputs.

## Operator Impact

- Generated alert-rule changes are now easier to review because expression, duration, label, annotation, and threshold changes appear in checked-in fixture diffs.
- The public Prometheus rules example is guarded against drifting away from the sample operator config.
- Representative generated alerts now have both positive and negative firing-behavior coverage.
- Site-specific Prometheus label edits still need validation against the operator's own Prometheus environment.
